# 派生（Fork）指南

本课程采用 MIT 许可证（MIT License）。您可以自由地派生（Fork）本项目并根据自身需求进行调整。以下是最佳实践指南。

## 面向团队

希望将其用作内部培训材料？请派生（Fork）并进行定制：

1. 派生（Fork）该代码仓库（Repository）
2. 移除团队不需要的阶段（Phases）
3. 添加公司专属的示例和数据
4. 在输出结果中集成内部工具
5. 保留署名信息——这有助于社区发展

## 面向学校与高校

希望将其用作课程教材？

1. 派生（Fork）该代码仓库
2. 将各阶段内容与学期教学进度对齐
3. 为练习添加评分标准（Rubrics）
4. 添加您自己的作业和考试
5. 考虑将改进内容贡献回上游（Upstream）仓库

## 面向训练营

运营付费训练营？在 MIT 许可证下这是完全允许的。

1. 根据学员批次的时间线进行派生（Fork）和结构调整
2. 添加视频内容、直播课程和导师指导环节
3. 代码和文档均可作为您二次开发的基础
4. 考虑赞助本项目或回馈贡献

## 面向其他编程语言

希望使用其他编程语言讲授本课程？

1. 派生（Fork）该代码仓库
2. 使用目标语言重新实现代码示例
3. 保留原有的课程结构和文档
4. 提交拉取请求（Pull Request），以便在主仓库的 README 文件中链接您的派生（Fork）版本

## 保持派生（Fork）版本同步更新

git fetch upstream
git merge upstream/main
