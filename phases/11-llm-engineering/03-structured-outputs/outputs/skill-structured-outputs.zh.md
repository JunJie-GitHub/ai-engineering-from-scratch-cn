# 结构化输出策略 (Structured Output Strategy)

在构建需要结构化数据的大语言模型（Large Language Model, LLM）应用时，请应用此决策框架。

## 何时使用各方法

**基于提示词的方法（Prompt-based，即“返回 JSON"）：** 仅适用于原型设计。适用于偶尔出现解析失败也可接受的内部工具。请添加带有重试机制的 `try/except` 块。切勿在生产流水线中使用。

**JSON 模式（JSON Mode，API 标志位）：** 当你需要保证输出为有效的 JSON，但模式（Schema）较为简单或灵活时使用。适用于在应用端验证数据结构的情况。支持平台：OpenAI、Anthropic（通过工具调用）、Google。

**模式模式（Schema Mode，约束解码 Constrained Decoding）：** 适用于生产系统，要求每次输出都必须严格匹配特定模式。可实现零解析失败、零模式违规。对于任何生产环境的数据提取或分类任务，应默认使用此方法。支持平台/库：OpenAI 结构化输出、Outlines、Guidance。

**函数调用 / 工具使用（Function Calling / Tool Use）：** 当模型需要自主选择调用哪个函数，而不仅仅是填充参数时使用。适用于存在多个模式且模型需自行选择合适模式的场景。在集成现有工具/函数基础设施时也应使用此方法。

**Instructor 库：** 当你希望跨任意提供商使用 Pydantic 验证并具备自动重试功能时使用。这是 Python 项目开发者体验（Developer Experience, DX）最佳的选择。它封装了 OpenAI、Anthropic、Google 以及开源模型。

## 各提供商特定指南

**OpenAI：** 使用 `response_format` 并设置类型为 `json_schema`。内置约束解码功能。可直接使用 Pydantic 模型。这是目前最可靠的结构化输出实现方案。

**Anthropic：** 使用工具调用（Tool Use）实现结构化输出。定义一个包含目标模式的单一工具。模型将返回符合该模式的工具调用参数。该方法可靠，但需遵循工具调用 API 模式。

**开源模型（vLLM, Ollama）：** 使用 Outlines 或 Guidance 进行约束解码。这些库会将 JSON 模式（JSON Schema）编译为有限状态机（Finite State Machine），在生成过程中屏蔽无效词元（Token）。需要在本地运行推理。

## 模式设计指南

1. 尽可能保持模式扁平化。嵌套对象超过 2 层会增加提取错误率。
2. 对分类字段使用枚举（Enum）。不要依赖模型自行“发明”正确的字符串。
3. 将含义模糊的字段设为必填（Required），并明确支持 `null` 值，而非设为可选。这能强制模型做出明确判断。
4. 为模式属性添加描述（Description）。模型会将这些描述视为指令来执行。
5. 除非必要，否则避免使用联合类型（Union Types，如 `oneOf`/`anyOf`）。它们会增加解码复杂度。
6. 为数值字段设置最小值/最大值（Minimum/Maximum）。可有效捕获模型幻觉（Hallucination）产生的极端值。
7. 在数组上使用 `minItems`/`maxItems` 限制，以防止输出为空或无限长。

## 常见失败模式及修复方案

- **模型将 JSON 包裹在 Markdown 代码块中**：从基于提示词的方法切换至 JSON 模式或模式模式
- **模式验证通过但事实错误**：在提取后增加“大语言模型作为裁判（LLM-as-a-Judge）”的验证步骤
- **枚举值不一致**：切换至约束解码或添加后处理归一化步骤
- **缺失可选字段**：将其改为必填，或在应用代码中添加默认值
- **提取速度极慢**：约束解码会增加 5-15% 的延迟，若对延迟敏感，请降低模式复杂度
- **包含多样化元素的大型数组**：对输入进行分块（Chunking），逐块提取后再合并结果

## 可靠性阶梯

| 方法 | 解析成功率 | 模式匹配率 | 配置耗时 |
|----------|-------------|-------------|-------------|
| 基于提示词的方法 | ~90% | ~80% | 1 分钟 |
| JSON 模式 | 100% | ~90% | 5 分钟 |
| 模式模式 | 100% | ~99% | 15 分钟 |
| 约束解码 | 100% | 100% | 30 分钟 |
| Instructor + 重试 | 100% | ~99.5% | 10 分钟 |
