# 推理技术选择指南

当需要大语言模型（Large Language Model, LLM）对问题进行推理时，应在编写提示词（Prompt）前先选定技术。该技术决定了推理架构，而提示词则负责填充具体内容。

## 快速决策树

1. 该任务是否为简单的事实查询或单步分类？
   - 是：使用 **零样本（Zero-shot）**。思维链（Chain-of-Thought, CoT）会增加成本且无法提升准确率。
   - 否：继续下一步。

2. 该任务是否需要多步推理（如数学、逻辑、规划）？
   - 是：使用 **思维链（Chain-of-Thought）**。继续至第 3 步。
   - 否：若格式重要则使用 **少样本（Few-shot）**，否则使用零样本。

3. 是否允许出现单次推理错误？
   - 是：使用 **少样本思维链（Few-shot CoT）**（单样本，温度值（Temperature）0.0）。
   - 否：使用 **自一致性（Self-Consistency）**（N=5，温度值 0.7）。继续至第 4 步。

4. 该问题是否为具有多条可能路径的搜索/规划问题？
   - 是：使用 **思维树（Tree-of-Thought, ToT）**。
   - 否：自一致性已足够。

5. 该任务是否需要外部信息或计算？
   - 是：使用 **ReAct**（推理 + 工具调用）。
   - 否：纯推理技术已足够。

## 技术矩阵

| 技术 | 准确率提升 | 成本倍数 | 延迟 | 最佳适用场景 |
|-----------|--------------|-----------------|---------|----------|
| 零样本（Zero-shot） | 基准 | 1x | ~1s | 简单任务、事实问答 |
| 少样本（Few-shot） | +5-15% | 1.2x | ~1s | 格式匹配、分类任务 |
| 零样本思维链（Zero-shot CoT） | +10-20% | 1.3x | ~1.5s | 快速提升推理能力 |
| 少样本思维链（Few-shot CoT） | +15-25% | 1.5x | ~2s | 数学、逻辑、多步任务 |
| 自一致性（Self-Consistency, N=5） | 较 CoT 提升 +2-5% | 5x | ~5s | 高风险推理任务 |
| 自一致性（Self-Consistency, N=10） | 较 N=5 提升 +1-2% | 10x | ~10s | 仅限关键决策 |
| 思维树（Tree-of-Thought） | 视任务而定 | 10-40x | ~30s+ | 搜索、规划、解谜 |
| ReAct | 视任务而定 | 3-10x | ~5-15s | 基于知识的任务 |
| 提示词链（Prompt Chaining） | 较单次提升 +5-10% | 2-5x | ~5-10s | 复杂多部分任务 |

## 模型专属指南

### GPT-4o / GPT-4.1
- 具备强大的基础推理能力。零样本思维链通常已足够。
- 使用 3 个示例的少样本思维链在 GSM8K 基准测试上可达 95% 的准确率。
- 自一致性带来的收益有限（95% 提升至 97%）——仅对关键任务具有性价比。
- 原生支持结构化输出，便于答案提取。

### Claude 3.5 Sonnet / Claude 3.7 Sonnet
- 极擅长遵循结构化提示词格式（如 XML 标签）。
- 使用 XML 分隔示例的少样本思维链效果最佳。
- 扩展思考功能（Claude 3.7）属于原生思维链——无需额外提示。
- 自一致性效果显著，因为 Claude 在温度值 0.7 下的推理路径具有良好的多样性。

### Llama 3.1/3.3 70B
- 从少样本思维链中获益最大（与零样本相比准确率差距更显著）。
- 推理任务推荐使用 N=5 的自一致性。
- 相比商业模型，需要更明确的格式指令。
- 在本地推理中运行思维树（ToT）成本较高——建议仅用于批处理场景。

### Gemini 2.5 Pro
- 开箱即具备强大的多步推理能力。
- 思考模式内置思维链，无需进行提示词工程（Prompt Engineering）。
- 少样本示例对格式一致性的帮助大于对准确率的提升。
- 超大上下文窗口（100 万 token）使得包含大量示例的少样本提示具备实际可行性。

## 反模式（Anti-Patterns）

**简单任务使用思维链**：询问“2+2 等于多少？让我们一步步思考”会浪费 Token。模型无需推理轨迹即可正确完成简单算术。思维链仅在步骤达到 3 步及以上时才发挥作用。

**温度值为 0.0 时使用自一致性**：所有 N 个样本将完全相同。必须使用大于 0 的温度值（推荐 0.5-0.8）以生成多样化的推理路径。

**所有场景均使用思维树**：思维树需要 O(b^d) 次大语言模型调用，其中 b 为分支因子，d 为深度。一个 b=3、d=3 的树最多需要 39 次调用。请仅将其保留给低成本技术无法解决的问题。

**使用错误示例的少样本提示 (Few-shot with bad examples)**：包含推理错误的示例会引导模型犯同样的错误。必须对每个示例进行严格验证。一个错误示例对模型准确率的负面影响，甚至可能超过完全不使用示例（零样本）。

**提取答案时格式不一致 (Extracting answers without a consistent format)**：自一致性 (Self-consistency) 机制需要跨样本比对答案。如果答案格式不统一（例如“$18”、“18美元”、“十八”），投票机制就会失效。务必强制规定输出格式为：“The answer is [number]。”

## 成本优化 (Cost Optimization)

对于每天处理 10,000 次查询的生产系统，按 GPT-4o 的定价（输入 $2.50/百万 token，输出 $10/百万 token）计算：

| 技术 | 平均每次查询 Token 数 | 每日成本 | 准确率 |
|-----------|-----------------|------------|----------|
| 零样本提示 (Zero-shot) | ~200 | ~$5 | 78% |
| 少样本思维链 (Few-shot CoT) | ~600 | ~$15 | 95% |
| 自一致性 (Self-Consistency, N=5) | ~3,000 | ~$75 | 97% |
| 思维树 (ToT, b=3, d=2) | ~6,000 | ~$150 | 视任务而定 |

对于大多数应用，最具成本效益的策略是：从少样本思维链 (Few-shot CoT) 开始。仅在对置信度较低的查询时，才引入自一致性 (Self-consistency)（即“构建它”章节中提到的升级模式）。

## 与提示链 (Prompt Chaining) 的集成

推理技术可与提示链 (Prompt Chaining) 组合使用：

**链步骤 1**（提取）：零样本提示 (Zero-shot)，温度参数 (Temperature) 0.0
**链步骤 2**（推理）：少样本思维链 (Few-shot CoT)，温度参数 0.0
**链步骤 3**（验证）：自一致性 (Self-consistency, N=3)，温度参数 0.7

该三步链的成本约为单次思维链 (CoT) 调用的 3 倍，但能有效拦截提取错误与推理错误，并通过验证步骤输出置信度评分。

## 何时超越提示工程 (Prompting)

如果你在提示工程 (Prompt Engineering) 上耗费的时间超过了编写应用代码的时间，建议考虑：

1. **微调 (Fine-tuning)**：如果你拥有 500 个以上的标注样本，且任务范围较窄
2. **DSPy 编译 (DSPy compilation)**：如果你希望实现提示词的自动优化
3. **智能体框架 (Agent frameworks)**：如果任务需要多轮工具调用（第 14 阶段）
4. **检索增强生成 (RAG)**：如果模型需要访问私有或最新知识（第 06-07 课）

提示技术是基础。它们适用于任何模型和任何提供商，且无需训练数据。但它们也存在局限性。明确何时该进阶到下一阶段，与掌握这些技术本身同等重要。
