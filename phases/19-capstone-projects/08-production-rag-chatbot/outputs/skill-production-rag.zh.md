给定一个受监管领域的语料库（如法律合同、临床试验方案、保险单或类似文档），部署一个聊天机器人。该机器人需提供可验证的引用来源，严格遵守角色与司法管辖区的访问策略，并支持漂移监控。

构建计划：

1. 使用 docling 或 Unstructured 解析语料库；将视觉信息丰富的文档路由至 ColPali 处理。输出带有角色和司法管辖区标签的文本块（chunks）。
2. 将稠密向量（使用 Voyage-3 或 Nomic-embed-v2 模型）索引至 pgvector + pgvectorscale；通过 Tantivy 实现稀疏 BM25 检索。
3. 搭建 LangGraph 对话智能体：检索（按角色与司法管辖区过滤，稠密+BM25 混合检索，倒数排名融合 RRF）、重排序（使用 bge-reranker-v2-gemma-2b 或 Voyage rerank-2）、生成合成（使用 Claude Sonnet 4.7 并启用提示词缓存）。
4. 组装具有稳定前缀的提示词：系统前言 -> 策略块 -> 重排序后的上下文 -> 用户查询。目标提示词缓存命中率达到 60-80%。
5. 安全护栏：输入输出端使用 Llama Guard 4；针对越域及违反策略的问题，配置 NeMo Guardrails v0.12 规则；输出端使用 Presidio 进行个人身份信息（PII）脱敏；后置过滤强制要求引用。
6. 构建包含 200 个问题的专家标注黄金数据集（含答案与引用）。评估指标包括精确引用匹配度、答案正确率以及 RAGAS 忠实度（faithfulness）。
7. 构建包含 50 个提示词的红队测试集（涵盖 PAIR、TAP、PII 提取、越域探测及跨司法管辖区探测）。
8. 使用 Arize Phoenix 漂移监控面板，每周跟踪检索归一化折损累计增益（nDCG）与引用忠实度；下降 5% 时触发告警。
9. Langfuse 成本报告：提示词缓存命中率、单次查询 Token 消耗、各阶段单次查询成本（$/query）。

评估标准：

| 权重 | 评估标准 | 测量方式 |
|:-:|---|---|
| 25 | RAGAS 忠实度 + 答案相关性 | 在 200 题黄金数据集上的在线评分 |
| 20 | 引用正确性 | 包含可验证来源锚点的答案占比 |
| 20 | 安全护栏覆盖率 | Llama Guard 4 通过率 + 越狱测试套件结果 |
| 20 | 成本/延迟工程优化 | 提示词缓存命中率、p95 延迟、单次查询成本（$/query） |
| 15 | 漂移监控面板 | 实时 Phoenix 面板及每周检索质量趋势 |

硬性否决项：

- 任何泄露跨司法管辖区数据的聊天机器人。角色与司法管辖区过滤必须在检索前强制执行，而非检索后。
- 破坏缓存前缀的合成提示词（例如在系统提示与上下文之间重新排序策略块）。这将彻底破坏缓存的经济效益。
- 未记录红队测试运行结果的安全护栏配置。
- 无引用的答案；或无可靠锚点验证的引用。

拒绝规则：

- 若语料库的每个文本块未标注司法管辖区标签，则拒绝在受监管领域部署。
- 拒绝使用专家标注的黄金数据集问题来训练检索模型。数据污染将破坏评估的可信度。
- 若 README 文件中未明确列出 SOC2/HIPAA/GDPR 适用性矩阵，则拒绝宣称系统“合规”。

交付物：一个代码仓库，需包含数据摄入流水线、LangGraph 对话智能体、200 题黄金数据集、50 提示词红队测试集、Phoenix 漂移监控面板、Langfuse 成本面板，以及一份说明文档。文档需列出你观察到的前三大引用断裂模式，并针对每种模式给出相应的检索或提示词修复方案。
