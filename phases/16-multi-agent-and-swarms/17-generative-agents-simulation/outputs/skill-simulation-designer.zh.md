针对需要智能体群体产生涌现行为（Emergent Behavior）的场景（如社会模拟、游戏 NPC、政策推演、市场动态），设计相应的模拟系统。

输出内容：

1. **群体规模与异质性（Population Size and Heterogeneity）。** N 个智能体；明确哪些共享基础模型（Base Model），哪些使用不同模型；提示词族（Prompt Families）；角色分布。Smallville 使用了 25 个具有个性化人设的同质化智能体；更大规模的群体则更适合采用异质性设计。
2. **记忆模式（Memory Schema）。** 每条记录的字段：`(ts, kind, content, importance, embedding_ref, source_ids)`。近因衰减常数（Recency-decay Constant）；重要性评分流程；相关性度量指标（使用嵌入模型 X 计算余弦相似度）。用于记忆压缩的保留策略。
3. **反思节奏（Reflection Cadence）。** 触发条件：未处理的重要性总和 > 阈值，或每 N 次观察后，或按固定时间步（Tick）触发。每次触发的反思次数。反思提示词模板。
4. **规划跨度（Plan Horizon）。** 天/小时/动作层级。明确哪些为强制层级，哪些为可选层级。修订触发条件：出现重要性 > 阈值且与当前活跃计划相矛盾的新观察。
5. **世界模型（World Model）。** 空间网格、社会关系图、资源约束。明确构成观察的要素（视线范围、对话、通知）。架构无法自行学习且必须显式编码的规范性约束（容量限制、非营业时间、私人空间）。
6. **初始目标（Seed Goals）。** 为哪些智能体分配何种初始优先级。可能产生竞争的重叠目标；应当共存的非竞争目标。
7. **预算（Budget）。** 每个智能体每时间步的大语言模型（LLM）调用次数（观察 + 检索 + 反思 + 规划 + 行动）。每个智能体每时间步的预期 Token 消耗。T 个时间步的总模拟成本。
8. **评估指标（Evaluation Metric）。** 可信度（人工评分）、目标达成率、协调事件计数、将违反空间规范的行为作为失败信号。

硬性拒绝条件（Hard Rejects）：

- 未显式编码空间/社会规范的设计。该架构会违反这些规范（如 Park 2023 中出现的“商店关门仍营业”、“单人浴室多人使用”等失败案例）。
- 采用可变记忆的设计。记忆必须为仅追加（Append-only）模式；修正操作应作为新条目添加。
- 每个时间步都执行反思的设计。这在预算上效率低下；反思计算成本高昂，触发机制应基于阈值。
- 大规模群体（N > 50）且缺乏记忆压缩策略的模拟。检索成本会随记忆流长度增加而上升。

拒绝规则（Refusal Rules）：

- 如果场景需要涌现的*任务执行*而非涌现的*社会行为*，建议改用主管/角色/原语模式（Supervisor / Roles / Primitives Patterns）（第 16 阶段 · 05-08）。Smallville 专用于社会模拟。
- 如果总预算允许每时间步少于 100 次 LLM 调用，建议采用 N = 3-5 且交互密集的设计，而非更大规模的群体。
- 如果场景无法从涌现中获益（如严格脚本化的任务），建议采用单智能体 + 工具（Single-agent + Tools）方案。

输出：一份单页设计简报。以单句摘要开头（“Smallville 风格模拟：15 个异质智能体，重要性总和 > 120 时触发反思，3 级规划跨度，带容量约束的空间网格，通过可信度与协调事件进行评估。”），随后列出上述八个部分。最后附上预期的涌现行为以及需要重点监控的前三种失败模式。
