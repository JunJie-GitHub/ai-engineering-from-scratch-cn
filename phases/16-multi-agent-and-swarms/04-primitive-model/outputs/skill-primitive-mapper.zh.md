给定一个多智能体框架（或使用了该框架的代码库），请生成四原语映射，以便读者能用一段话理解该框架。

请生成：

1. **智能体 (Agent) 定义。** 智能体如何构建？需要哪些参数？携带何种状态？请指明具体的类或工厂函数。
2. **交接 (Handoff) 机制。** 它采用三种交接模式中的哪一种——函数返回、图边还是发言者选择？如果是混合模式，以哪种为主？请展示触发一次交接的最简代码。
3. **共享状态 (Shared State) 模型。** 采用完整消息池还是投影视图？内存驻留还是持久化（带检查点）？是否支持并发写入的线程安全？由谁负责冲突协调？
4. **编排器 (Orchestrator) 类型。** 静态、大语言模型 (LLM) 选择、交接驱动还是队列驱动？若为 LLM 选择，默认使用哪个模型？若为静态，图结构是有环图还是有向无环图 (DAG)？
5. **跨轴权衡 (Cross-axis Tradeoffs)。** 各用一句话说明：确定性、可扩展性上限、可调试性、典型故障模式。

硬性拒绝条件：

- 任何声称某抽象概念为“全新”却未能证明其无法归约为四大原语之一的映射。若无法归约，请精确指出差异所在，而非凭空捏造第五个原语。
- 仅引用营销文档的框架对比。必须引用该框架代码仓库或官方示例库中的具体代码实例。
- 诸如“框架 X 更适合智能体”的表述，却未指明该框架优化了哪个原语。

拒绝规则：

- 若框架为闭源，且公开文档未暴露智能体-交接-状态-编排器接口，请声明在缺乏内部实现细节的情况下无法完成映射。
- 若用户提供的是代码库而非现成框架（即手写智能体），请映射其自定义实现，并标出设计薄弱的原语。
- 若框架发布于 2024 年之前（如原版 AutoGen v0.2、Swarm 之前版本）且已停止维护，请附一行说明，指出其继任者是否保留了该映射关系。

输出：一份单页框架简报。以单句摘要开头（例如：“框架 X 将交接固定为图边，并通过归约器暴露共享状态。”），随后是上述五个部分，最后以一段结语收尾，指明该框架的原语最适合哪类生产级项目。
