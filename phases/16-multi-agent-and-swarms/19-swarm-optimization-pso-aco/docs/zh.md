# 大语言模型的群体优化（PSO, ACO）

> 受生物启发的优化算法（Bio-inspired optimization）正在大语言模型（LLM）领域迎来复兴。**LMPSO**（arXiv:2504.09247）采用粒子群优化（PSO），其中每个粒子的“速度”被表示为提示词（prompt），并由大语言模型生成下一个候选解；该方法在结构化序列输出（如数学表达式、程序代码）上表现优异。**Model Swarms**（arXiv:2410.11163）将每个大语言模型专家视为模型权重流形（model-weight manifold）上的一个PSO粒子，在仅使用200个样本的情况下，于9个数据集上对比12个基线模型，报告了**13.3%的平均性能提升**。**SwarmPrompt**（ICAART 2025）将PSO与灰狼优化算法（Grey Wolf Optimizer）相结合，用于提示词优化。**AMRO-S**（arXiv:2603.12933）受蚁群优化（ACO）启发，采用信息素（pheromone）机制专门用于多智能体大语言模型的路由（routing）决策——实现了**4.7倍的加速**，提供可解释的路由依据，并通过质量门控的异步更新机制将推理与学习过程解耦。本课程将在提示词参数空间上实现PSO，在智能体路由上实现ACO，并评估这些经典算法为何契合大语言模型时代，以及在何种场景下不再适用。

**类型：** 学习 + 实战
**编程语言：** Python（标准库）
**前置知识：** 第16阶段 · 09（并行群体网络），第16阶段 · 14（共识与BFT）
**预计耗时：** 约75分钟

## 问题

你有一个在任务评估中得分为62%的提示词，希望对其进行优化。最直接的做法是进行无梯度（gradient-free）的手动调整，但这种方法难以扩展。强化学习需要奖励信号以及足够的轨迹采样（rollout）来进行训练。而通过提示词进行反向传播实际上并不可行——因为提示词是离散字符串，而非可微参数。

经典的受生物启发的优化算法——用于连续搜索空间的PSO，以及用于路径选择的ACO——正是为这种场景设计的：无需梯度、基于种群（population-based）、单次评估成本低。将它们与大语言模型结合用于无梯度搜索步骤，就能得到一个出乎意料地实用的优化器。

同样的模式也适用于多智能体系统中的智能体*路由*。ACO风格的信息素轨迹会记录哪种智能体在何种任务类型上表现最佳，使路由器能够利用该轨迹进行决策，同时通过信息素衰减机制确保旧路径可以被重新探索。

## 概念

### PSO 概念回顾（Kennedy & Eberhart 1995）

粒子群优化（Particle Swarm Optimization）：在连续搜索空间中维护一群粒子。每个粒子具有位置 `x_i` 和速度 `v_i`。每次迭代执行以下操作：

v_i <- w * v_i + c1 * r1 * (p_best_i - x_i) + c2 * r2 * (g_best - x_i)
x_i <- x_i + v_i
evaluate fitness(x_i)
update p_best_i if improved
update g_best if global best

其中，`p_best` 表示粒子自身的历史最优位置，`g_best` 表示整个群体的全局最优位置，`w, c1, c2` 分别代表惯性（inertia）权重、个体认知权重和社会学习权重，`r1, r2` 为随机因子。

### 应用于 LLM 输出的 PSO —— LMPSO

arXiv:2504.09247 将 PSO 适配于大语言模型（Large Language Model）生成的结构化输出（如数学表达式、代码程序）。每个粒子代表一个候选输出。速度被转化为一个*提示词（prompt）*，用于描述如何将当前输出向个体最优或全局最优方向调整。LLM 根据该速度提示词生成新的输出。速度的“惯性”则通过类似“进行小幅渐进式修改”的提示词来实现。

该方法在以下场景中表现良好：
- 输出具有结构化特征（可解析、可评估）。
- 适应度（fitness）评估可自动化完成（如运行测试用例、算术求值）。
- 种群规模较小（约 10-30 个粒子），从而确保 LLM 调用总量处于可控范围。

当适应度评估需要人工审核时，该方法效果不佳——因为单次迭代的成本将变得难以承受。

### 模型群（Model Swarms）

arXiv:2410.11163 将 PSO 从输出层迁移至*模型*层。每个“粒子”对应一个专家 LLM（及其参数）。群体通过无梯度更新（gradient-free update）将参数推向集体最优。实验报告指出：在 9 个数据集上，相较于 12 个基线模型平均提升 13.3%，且每次迭代仅需 200 个样本实例。

其核心洞察在于：LLM 专家模型在共享的参数流形（parameter manifold）（如适配器权重、LoRA 增量）中本就彼此邻近。在此低维子空间上应用 PSO 既经济又高效。

### ACO 概念回顾（Dorigo 1992）

蚁群优化（Ant Colony Optimization）：蚂蚁在图结构上遍历；每条路径都留有信息素轨迹。蚂蚁的移动概率由信息素浓度加权决定。成功完成任务的蚂蚁会根据解的质量按比例释放信息素。信息素会随时间自然衰减。

### AMRO-S —— 用于智能体路由的 ACO

arXiv:2603.12933 将 ACO 应用于多智能体路由（multi-agent routing）。每种任务类型被视为一个“目的地”；每个智能体则是一条潜在路由。信息素会强化那些能产出高质量输出的路由。其主要贡献包括：

- **可解释的路由依据。** 信息素浓度是一种人类可读的信号。
- **质量门控的异步更新。** 仅当质量检查通过后才会更新信息素，从而将推理过程与学习过程解耦。
- 在多智能体路由基准测试中实现 **4.7 倍加速**。

质量门控机制至关重要：若缺少该机制，响应迅速但结果错误的智能体会不断累积信息素，导致系统锁定在劣质路由上。

### 何时在 LLM 场景中使用 PSO / ACO

**适用 PSO 的场景：**
- 搜索空间为连续型，或可映射为连续参数（如提示词嵌入向量、LoRA 权重、数值型生成参数）。
- 适应度评估成本低且可自动化。
- 种群规模可控制在较小范围（10-30）。

**适用 ACO 的场景：**
- 面临路由或路径选择问题。
- 决策具有时间累积强化效应（同类任务会反复出现）。
- 需要为路由决策提供可解释的依据。

**两者均不适用的场景：**
- 适应度评估依赖人工审核（单次迭代成本过高）。
- 搜索空间为离散组合型，且超出 PSO 的覆盖范围（此时应改用遗传算法）。
- 实时决策对延迟有严格要求（相较于单次启发式方法，PSO/ACO 的收敛速度较慢）。

### 为何仿生算法依然具有优势

基于梯度的方法依赖可微信号。而 LLM 的输出与路由决策通常不具备直接的可微性。伪梯度方法（如基于强化学习的路由器、DPO 风格的提示词调优器）虽然有效，但需要高昂的训练成本。

PSO 和 ACO 仅需一个*评估器（evaluator）*函数。只要你能对候选输出或路由决策进行打分，就能在该空间上进行优化。这大幅降低了算法的应用门槛。

### 实际限制

- **种群预算。** 计算公式为：N 个粒子 × T 次迭代 × 单次评估成本。以 LLM 评估成本约 0.02 美元/次为例，一个包含 20 个粒子的 PSO 运行 50 次迭代，成本约为 20 美元。需提前规划预算。
- **探索与利用的权衡。** 信息素衰减率与 PSO 惯性权重之间存在权衡关系；衰减过快会导致遗忘已有解；衰减过慢则容易陷入早期局部最优。
- **灾难性漂移。** 若适应度地形（fitness landscape）发生偏移（如数据分布变化），两种算法均可能出现先收敛后发散的情况。需密切监控最优适应度的稳定性。

## 构建它

`code/main.py` 实现了：

- `LMPSO` — 基于数值型提示词参数（如温度、top_k 权重）的粒子群优化 (Particle Swarm Optimization, PSO)。每个粒子的“大语言模型 (Large Language Model, LLM) 生成”过程被模拟为一个脚本化的适应度函数 (fitness function)。该脚本运行算法 30 次迭代，并展示 g_best 的收敛 (convergence) 情况。
- `AMRO_S` — 蚁群优化 (Ant Colony Optimization, ACO) 风格的路由算法。包含 3 个智能体 (agent)、4 种任务类型、信息素矩阵 (pheromone matrix) 以及 100 个已路由任务。随时间打印（任务类型 → 智能体选择）的分布情况，以展示信息素轨迹的形成过程。
- 对比：在相同任务流上对比随机路由与 ACO 路由。测量质量与延迟。

运行：

python3 code/main.py

预期输出：
- LMPSO：经过 30 次迭代，g_best 适应度从随机水平提升至接近最优。
- AMRO-S：信息素表会针对每种任务类型稳定收敛至正确的智能体；在质量方面，ACO 路由比随机路由高出约 30-40%，同时降低了延迟（重试次数减少）。

## 使用它

`outputs/skill-swarm-optimizer.md` 可帮助你在解决大语言模型 / 智能体优化问题时，在粒子群优化 (PSO)、蚁群优化 (ACO)、遗传算法 (Genetic Algorithms) 和基于梯度的优化器 (Gradient-based Optimizers) 之间做出选择。

## 部署与交付

- **从小规模开始。** 设置 10-20 个粒子，20-50 次迭代。仅当收敛曲线显示出明确收益时，再扩大规模。
- **记录每次迭代的信息素或 g_best。** 如果没有轨迹记录，调试群体优化器将非常痛苦。
- **设置质量门禁 (quality gate) 更新。** 尤其是对于 ACO 路由：响应快但结果错误的智能体绝不能累积信息素。
- **在分布偏移 (distribution shift) 时重置衰减。** 当评估数据分布发生变化时，陈旧的信息素会失效；此时应重置信息素或临时将衰减率加倍。
- **限制单次迭代成本。** 输出单次迭代成本指标。如果 PSO 每次迭代成本高达 500 美元却仅带来 0.5% 的收益，则不具备上线价值。

## 练习

1. 运行 `code/main.py`。观察 LMPSO 的收敛情况。将种群大小分别设置为 5、10、20、50。在多大尺寸时，收敛时间会趋于饱和？
2. 实现一个“灾难性漂移 (catastrophic drift)”实验：在第 30 次迭代后更改适应度函数。PSO 的适应速度有多快？重置 `p_best` 是否有帮助？
3. 为 AMRO-S 添加质量门禁：仅在评估得分 > 0.7 的运行中沉积信息素。与未设置门禁的版本相比，这会如何改变收敛过程？
4. 阅读 LMPSO 论文 (arXiv:2504.09247)。将论文中的“将速度作为提示词”映射回你的数值型速度。在模拟过程中丢失了什么，又保留了什么？
5. 阅读 AMRO-S 论文 (arXiv:2603.12933)。实现解耦的“推理快速通道 (inference fast-path)”并配合异步信息素更新。在持续负载下，这会如何改变系统延迟？

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|----------------|------------------------|
| PSO | “粒子群优化” (Particle Swarm Optimization) | Kennedy 与 Eberhart 于 1995 年提出。一种基于种群且无需梯度的优化器 (gradient-free optimizer)。 |
| ACO | “蚁群优化” (Ant Colony Optimization) | Dorigo 于 1992 年提出。通过信息素轨迹 (pheromone trails) 进行路径/路线优化。 |
| LMPSO | “结合大语言模型生成的 PSO” | arXiv:2504.09247。将提示词 (prompt) 作为速度向量，由大语言模型 (LLM) 生成候选解。 |
| Model Swarms | “基于专家权重的 PSO” | arXiv:2410.11163。在模型参数子空间 (model parameter subspace) 上进行无梯度更新。 |
| AMRO-S | “用于智能体路由的 ACO” | arXiv:2603.12933。在任务类型 × 智能体维度上构建信息素矩阵。 |
| p_best / g_best | “个体/全局最优” | 单个粒子及整个种群迄今为止找到的最优解。 |
| Pheromone | “路由记忆” | 边上的强度值；随时间衰减；根据解的质量进行沉积。 |
| Quality-gated update | “仅从优质运行中学习” | 信息素沉积需以质量检查 (quality check) 为前提条件。 |
| Catastrophic drift | “分布偏移” | 适应度景观 (fitness landscape) 发生变化；旧的个体最优解和信息素失效。 |
