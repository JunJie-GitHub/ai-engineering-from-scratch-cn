针对需要大语言模型（Large Language Model, LLM）进行推理（如研究、摘要、规划、分类）的服务器端算法，设计一种基于 MCP 采样（Sampling）的实现方案。

产出内容：

1. 循环结构。为每一轮采样编号，说明提示词（Prompt）的结构，以及预期的输出类型。
2. 每轮的 `modelPreferences`。为每一轮分配成本/速度/智能程度的权重（总和为 1.0）。例如，“挑选文件”轮次偏向成本，“综合归纳”轮次偏向智能程度。
3. 速率限制（Rate Limit）。为每次调用设置 `max_samples_per_tool`；并说明该数值的设定依据。
4. 安全钩子（Safety Hook）。明确客户端应在何处显示确认对话框，以及拒绝路径（Refusal Path）的具体行为。
5. SEP-1577 包含项。决定是否在采样过程中使用工具（Tool）；若使用，需标记漂移风险（Drift Risk）并指定工具列表。

硬性拒绝条件：
- 任何未设置速率限制的循环。存在循环炸弹（Loop Bomb）和资源窃取风险。
- 任何将 `includeContext: "allServers"` 设置为该值的循环。存在跨服务器数据泄露风险。
- 任何服务器要求客户端生成内容，并在未经用户确认的情况下将其直接作为工具输入反馈的循环。存在混淆代理（Confused Deputy）攻击向量风险。

拒绝规则：
- 若服务器已具备自有 LLM 凭证，需确认是否确实需要采样；直接调用可能更为简便。
- 若用例仅为单次（One-shot）工具调用，则拒绝设计采样循环；采样适用于多轮推理场景。
- 若用户要求设计向终端用户隐藏意图的采样循环，则必须明确拒绝（隐蔽采样/Covert Sampling）。

输出要求：一份单页设计文档，包含循环步骤、每轮的 `modelPreferences`、速率限制及安全清单。文末需附加说明，标记该设计中涉及 SEP-1577（采样内使用工具）的任何相关漂移风险。
