# Qwen-VL 系列与动态帧率（Dynamic-FPS）视频

> Qwen-VL 系列——包括 Qwen-VL（2023）、Qwen2-VL（2024）、Qwen2.5-VL（2025）和 Qwen3-VL（2025）——是 2026 年最具影响力的开源视觉语言模型（Vision-Language Model）谱系。每一代模型都做出了一项决定性的架构选择，并在十二个月内被整个开源生态所效仿：通过 M-RoPE（多维旋转位置编码）实现原生动态分辨率、具备绝对时间对齐的动态帧率（Dynamic-FPS）采样、视觉变换器（Vision Transformer）中的窗口注意力机制（Window Attention），以及结构化的智能体（Agent）输出格式。到了 Qwen3-VL，这套配方已趋于稳定：采用支持原生宽高比输入的 2D-RoPE-ViT 编码器、通过多层感知机（Multilayer Perceptron）投影层接入大型 Qwen3 语言基座，以及将光学字符识别（Optical Character Recognition）、视觉定位（Grounding）和智能体行为作为一等公民目标的训练阶段。本教程将按时间顺序梳理该系列的发展脉络，帮助你理解每一个设计参数为何如此设定。

**类型：** 学习
**语言：** Python（标准库，M-RoPE 编码器 + 动态帧率采样器）
**前置条件：** 第 12 阶段 · 06（patch-n'-pack）
**时长：** 约 120 分钟

## 学习目标

- 计算 M-RoPE 的三轴旋转（时间轴、高度轴、宽度轴），并解释为何三者缺一不可。
- 为视频选择合适的动态帧率（Dynamic-FPS）采样策略，并权衡每秒令牌数（Tokens-per-second）与事件检测准确率之间的关系。
- 按顺序列出 Qwen-VL 的四次代际升级，并说明每次升级带来的能力突破。
- 实现类似 Qwen2.5-VL 的 JSON 智能体输出格式，并从视觉语言模型响应中解析结构化的工具调用（Tool Calls）。

## 问题背景

Qwen-VL 于 2023 年 8 月发布，旨在直接对标 LLaVA-1.5 和 BLIP-2。Qwen 团队瞄准的差距主要体现在三个方面：分辨率、视频处理以及结构化输出。

分辨率：LLaVA-1.5 的运行分辨率为 336x336。这对于普通照片尚可，但面对中文发票或密集的表格截图则完全无能为力。Qwen-VL 的首项创新是将分辨率提升至 448x448，并引入带视觉定位的边界框（Bounding Box）输出，使模型能够精准“指向”图像中的具体目标。

视频处理：Video-LLaMA 采用逐帧编码器堆叠的方式，并将结果输入大语言模型（Large Language Model）。这种方法适用于短视频片段，但对于时长数分钟、且时间轴本身即为关键信号的视频则力不从心。Qwen 团队的目标是打造一个能够真正理解时间维度的单一编码器。

结构化输出：LLaVA 仅能生成自由格式的文本，而智能体需要的是 JSON 格式。Qwen-VL 在训练时明确采用了 JSON 输出格式，甚至将边界框坐标也作为文本纳入其中。

Qwen-VL 的每一代迭代，都在持续深化这三个维度中的某一个。

## 核心概念

### Qwen-VL（2023 年 8 月）

第一代：采用 OpenCLIP ViT-bigG/14 作为编码器（25 亿参数），兼容 LLaMA 的 Q-Former（单步处理，256 个查询向量），基于 Qwen-7B。主要贡献：

- 448x448 分辨率（当时开源视觉语言模型 VLM 的业界最优水平 SOTA）。
- 视觉定位（Grounding）：使用带有显式坐标标记（coordinate-token）输出的图文对进行训练。例如：“猫位于 `<box>(112, 204), (280, 344)</box>`”。
- 从一开始就支持中英双语训练。

当时的基准测试表现：在英文任务上与 GPT-4V 相当，在中文任务上占据主导地位。而视觉定位监督（grounding supervision）才是其真正的亮点。

### Qwen2-VL（2024 年 9 月）—— M-RoPE 与原生分辨率

Qwen2-VL 用原生动态分辨率的 ViT 编码器取代了“固定分辨率 + Q-Former”的架构堆栈。关键改进如下：

- 原生动态分辨率。ViT 可接受任意宽高（HxW）能被 28 整除的图像（14x14 的图像块 patch 经过 2 倍空间合并）。例如 1120x672 的图像（合并后为 40x24 个图像块）会生成 960 个视觉标记（visual tokens）。无需缩放（resize）、无需分块（tiling）、无需缩略图（thumbnail）。
- M-RoPE（多模态旋转位置编码 Multimodal RoPE）。每个标记携带三维位置信息 `(t, h, w)` 而非一维。对于图像 `t=0`，对于视频 `t = frame_index`（帧索引）。RoPE 按每个轴对应的频率对查询/键（query/key）向量进行旋转。无需位置嵌入表（positional embedding table）。
- MLP 投影器。摒弃 Q-Former，直接在合并后的图像块标记上使用两层多层感知机（MLP）。
- 支持动态帧率（FPS）的视频。视频默认以 1-2 FPS 采样，但模型可接受任意数量的帧。

结果：Qwen2-VL-7B 在多项多模态基准测试中与 GPT-4o 持平，并在 DocVQA 上超越后者（94.5 vs 88.4）。架构的变革是此次性能跃升的决定性因素。

### Qwen2.5-VL（2025 年 2 月）—— 动态帧率与绝对时间

Qwen2.5-VL 的重大转变在于视频处理。动态帧率不仅仅是“按需采样更多帧”。该论文将其形式化为：

- 绝对时间标记（Absolute time tokens）。不再使用位置索引（第 0、1、2 帧……），而是使用实际时间戳。例如：“在 0:04，猫跳了起来。”模型会看到与帧标记交错排列的 `<time>0.04</time>` 标记。
- 动态帧率。慢速画面以 1 FPS 采样，动作画面以 4+ FPS 采样。由用户或训练者选择，M-RoPE 会自适应调整。
- ViT 中的窗口注意力机制（Window attention）。空间注意力采用窗口化（块内局部计算）以提升吞吐量；每隔几层使用一次全局注意力。
- 显式 JSON 输出格式。使用工具调用（tool-call）数据进行训练：`"{\"tool\": \"click\", \"coords\": [380, 220]}"`。开箱即用，具备智能体（Agent）就绪能力。
- MRoPE-v2 缩放机制。位置编码随最大输入尺寸进行缩放，确保 10 分钟长的视频不会耗尽频率范围。

基准测试表现：Qwen2.5-VL-72B 在大多数视频基准测试中击败 GPT-4o，在文档理解任务上与 Gemini 2.0 持平，并刷新了开源模型在 GUI 视觉定位（GUI grounding）上的 SOTA 记录（ScreenSpot：准确率 84%，而 GPT-4o 为 38%）。

### Qwen3-VL（2025 年 11 月）

Qwen3-VL 是一次渐进式升级，侧重于巩固而非重构：采用更大的大语言模型（LLM）主干网络（Qwen3-72B）、扩展训练数据、提升光学字符识别（OCR）能力，并通过 Qwen3 的“思考模式”增强推理能力。ViT 和 M-RoPE 架构得以保留。该论文的重点在于数据与训练策略的优化，而非架构变更。

演进脉络的启示：到 2025 年，Qwen-VL 的架构已趋于稳定。后续代际的演进主要在于算力与数据规模的扩展，而非基础原语（primitives）的更迭。

### M-RoPE 的数学原理

经典 RoPE 使用成对坐标，根据位置 `m` 对维度为 `d` 的查询向量 `q` 进行旋转：

q_rot[2i]   = q[2i]   * cos(m * theta_i) - q[2i+1] * sin(m * theta_i)
q_rot[2i+1] = q[2i]   * sin(m * theta_i) + q[2i+1] * cos(m * theta_i)
theta_i     = 10000^(-2i/d)

M-RoPE 将隐藏维度划分为三个频段。假设 `d = 96`，将 32 个维度分配给时间轴，32 个分配给高度轴，32 个分配给宽度轴。每个频段根据其对应轴的位置进行独立旋转。例如，位于 `(t=5, h=10, w=20)` 的图像块，其三个频段将分别应用旋转矩阵 `R_t(5)`、`R_h(10)` 和 `R_w(20)`。

文本标记使用 `t = text_index, h = 0, w = 0`（或归一化选择）以保持兼容性。视频帧使用 `t = frame_time, h = row, w = col`。单张图像使用 `t = 0`。

优势：单一的位置编码即可统一处理文本、图像和视频，无需分支代码或不同的位置表。

### 动态帧率（Dynamic-FPS）采样逻辑

给定一段时长为 `T` 秒的视频和目标标记预算 `B`：

1. 计算可承受的最大帧率：`fps_max = B / (T * tokens_per_frame)`。
2. 从 `{1, 2, 4, 8}` 中选择一个满足 `fps <= fps_max` 的目标帧率。
3. 若画面运动剧烈（基于光流启发式算法 optical-flow heuristic 或用户明确要求），则选择较高帧率；若运动平缓，则选择较低帧率。
4. 按选定帧率进行均匀采样；在帧之间插入 `<time>t</time>` 标记。

Qwen2.5-VL 在训练阶段隐式学习了该逻辑；在推理阶段，用户可通过 `fps` 参数进行控制。例如，一段 60 秒的动作序列以 4 FPS 采样，每帧 81 个标记，总计 19440 个标记，完全可在 32k 上下文窗口内轻松处理。

### 结构化智能体输出

Qwen2.5-VL 的智能体训练明确针对结构化工具调用：

{
  "tool": "mouse_click",
  "coords": [1024, 512],
  "button": "left",
  "modifier": null
}

解析过程是确定性的：直接对模型输出执行 `JSON.parse`。相比之下，自由文本格式如“点击 (1024, 512)”需要依赖正则表达式并处理歧义。正是这一转变，使得 Qwen2.5-VL 在 ScreenSpot 上的得分从 Qwen2-VL 的 55% 跃升至 84%。

## 使用它

`code/main.py` 实现了：

- 针对混合了文本、图像块（image patches）和视频帧的打包序列（packed sequence）的 M-RoPE（Multimodal Rotary Position Embedding）位置计算。
- 动态帧率（Dynamic-FPS）采样器：根据（时长、预算、运动等级）选择帧率并输出帧时间戳。
- 一个示例级（toy）的 Qwen2.5-VL JSON 输出解析器，用于处理包含坐标字段的工具调用（tool-call）响应。

运行该脚本，然后在一段 5 分钟的视频中将固定帧率替换为动态帧率，亲自感受两者的差异。

## 关键术语

| 术语 | 通俗说法 | 实际含义 |
|------|-----------------|------------------------|
| M-RoPE | “多模态 RoPE” | 在隐藏维度中包含时间、高度和宽度频段的 3D 旋转位置编码（3D rotary position embedding） |
| Dynamic FPS | “智能采样” | 根据视频的运动程度、时长和 token 预算为每个视频单独选择的帧采样率 |
| Absolute time token | “时间戳 token” | 交错插入序列中的 `<time>t</time>`，使模型能够感知实际秒数而非帧索引 |
| Window attention | “局部注意力” | 为提升速度而将空间自注意力限制在小窗口内；并周期性加入全局注意力 |
| Structured agent output | “JSON 模式” | 通过训练数据监督，教导视觉语言模型（VLM）输出包含坐标和工具名称的可解析 JSON |
| min_pixels / max_pixels | “分辨率边界” | Qwen2.5-VL 针对每次请求的控制参数，用于限制总像素数，从而控制 token 数量 |
| Grounding | “指哪儿打哪儿” | 将边界框坐标作为文本 token 输出；自 Qwen-VL v1 起开始使用 |
