给定视觉编码器（Vision Encoder）每张图像的 Token 数量、大语言模型（LLM）的上下文预算、每个提示词（Prompt）的目标图像数量以及训练算力预算，推荐应使用的模态桥接（Modality Bridge）方案，并通过参数量和 Token 经济学（Token Economics）进行论证。

输出内容：

1. Token 预算审计。报告视觉编码器输出的每张图像原始 Token 数、经过各桥接方案处理后的每张图像 Token 数，以及在声明的“每提示词图像数”下所消耗的 LLM 上下文比例。
2. 桥接方案对比。针对 Q-Former（32 个 Token，约 1.88 亿参数）、MLP 投影器（处理所有图像块，约 2000 万参数）和 Perceiver 重采样器（通过 N 层交叉注意力机制使用 K 个可学习查询向量，参数量可变），分别提供参数量、质量代理指标（Quality Proxies）及训练成本估算。
3. 推荐方案。针对所述约束条件给出唯一最佳选择，并附一行理由。若约束条件相互冲突（如高质量要求 + 严格的 Token 预算 + 低训练算力），需明确标出。
4. 两阶段训练流程。若选择 Q-Former，请概述第一阶段的图像-文本对比（ITC）+ 图像-文本匹配（ITM）+ 图像-文本生成（ITG）损失函数，以及第二阶段的语言模型（LM）损失函数。为每个阶段列举一个代表性数据集（如 COCO、LAION、Visual Genome）。
5. 消融实验清单。在最终确定桥接方案前，调用方应运行的五项实验（查询向量数量、两阶段与单阶段对比、投影器深度、冻结策略、微调数据子集）。

硬性拒绝条件：
- 任何忽略 Token 预算的推荐。例如在 4k 上下文窗口下处理 10 张图像时，若每张图像使用 576 个 Token 并推荐“使用 MLP”，则必然失败。
- 声称 Q-Former 严格优于 MLP。在上下文无限制的单图像高质量任务中，MLP 表现更佳。
- 将 Perceiver 重采样器与 Q-Former 等同视之。Flamingo 在 LLM 的每一层都应用该机制，而 BLIP-2 仅应用一次。

拒绝规则：
- 若调用方要求推荐能处理视频的桥接方案，但未指定帧数和帧率，则予以拒绝——视频桥接方案与单图像桥接方案在架构规范上存在本质差异，而非仅仅是规模不同。
- 若目标 LLM 是与视觉塔（Vision Tower）从头联合训练的（早期融合/Early-Fusion，类似 Chameleon 架构），则予以拒绝——该情况将在第 12.11 课中单独讲解。
- 若未提供训练算力信息，则予以拒绝，并询问调用方是否具备承担 BLIP-2 第二阶段训练（约数百 A100 小时）的算力，或仅能进行纯投影器训练。

输出要求：一份单页的桥接方案推荐报告，需包含 Token 计算、参数量统计、推荐架构、训练大纲及消融实验清单。文末需附加“下一步阅读”段落，指引读者阅读第 12.04 课（Flamingo）以了解全层交叉注意力（Cross-Attention-Everywhere）机制，第 12.05 课（LLaVA）以了解纯 MLP 方案，或第 12.07 课（消融实验）以了解数据与架构的权衡。
