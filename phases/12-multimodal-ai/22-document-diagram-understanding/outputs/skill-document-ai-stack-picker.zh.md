给定一个文档AI（Document AI）项目（领域：发票 / 科学论文 / 表单 / 混合；规模：每日页数；质量门槛；合规需求），选择一套技术栈并生成参考配置。

输出内容：

1. 技术栈选择。阶段一（OCR流水线 + LayoutLMv3）、阶段二（Donut / Nougat 免OCR模型）、阶段三（VLM原生），或混合架构。
2. 单页成本估算。所选技术栈下的Token数量与延迟。
3. 准确率预期。DocVQA + ChartQA + 领域特定基准测试。
4. 手写体处理策略。对成本不敏感的场景采用VLM原生方案；大规模场景采用专用TrOCR + 路由机制。
5. 数学公式 / LaTeX 输出。科学论文使用Nougat；其他场景使用VLM。
6. 合规兜底方案。采用混合架构并附带交叉核对审计日志。

硬性否决条件：
- 在未进行成本分析的情况下，为每日处理量超过100万页的场景推荐VLM原生方案。每页2576像素分辨率下的Token成本极为高昂。
- 为受监管的工作流推荐缺乏审计路径的单模型解决方案。
- 声称Nougat能够处理扫描版发票。该模型并不具备此能力——它是专为科学论文设计的专用模型。

拒绝规则：
- 若规模超过每日1000万页，拒绝阶段三方案，推荐阶段一方案，并将阶段三作为抽样验证器。
- 若领域以手写体为主，拒绝OCR流水线方案，推荐VLM原生 + 手写体专用模型（TrOCR）。
- 若公式需要高保真LaTeX输出，则必须在流程中引入Nougat。

输出要求：一页纸方案，涵盖技术栈、成本、准确率、手写体处理、数学公式及合规方案。文末附上参考文献：arXiv 2308.13418 (Nougat)、2204.08387 (LayoutLMv3)、2111.15664 (Donut)。
