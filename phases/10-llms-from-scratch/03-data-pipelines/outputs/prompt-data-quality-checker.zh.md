# 大语言模型（LLM）预训练数据质量检查器

在构建或审计大语言模型（LLM）预训练数据流水线（data pipeline）时，请使用此框架在数据输入模型前捕获潜在问题。

## 流水线输出中的危险信号

**去重（deduplication）步骤移除的网页数据不足 20%。** Common Crawl 数据集通常包含 30-40% 的重复内容。如果你的去重步骤移除比例低于 20%，说明你的最小哈希（MinHash）参数设置过于保守或阈值过高。请检查：连续词序列（shingle）大小 k、哈希函数数量、局部敏感哈希（LSH）分带数量、杰卡德（Jaccard）阈值。

**压缩比低于 2.0 字符/词元（chars/token）。** 这意味着你的分词器（tokenizer）切分过于激进。可以通过增加合并操作（merges）次数重新训练、扩大词表（vocabulary）大小，或检查预分词（pre-tokenization）步骤是否在不必要的情况下将文本过度碎片化。

**压缩比高于 6.0 字符/词元。** 你的分词器学习到了非常特定领域的合并规则，可能缺乏泛化能力。这对于领域专用模型来说可以接受，但对于通用模型而言是一个警告信号。

**序列利用率低于 90%。** 填充（padding）过多。要么是你的文档过短（应过滤它们或增加最小文档长度），要么是你的序列打包（sequence packing）效率低下（应从简单填充切换为多文档打包）。

**词表利用率低于 50%。** 你的词表中超过一半的词元在当前语料库中未被使用。要么是词表对于你的领域来说过大，要么是分词器是在差异极大的数据上训练的。

## 质量过滤器校准

在流水线的每个阶段，随机抽取 1,000 份文档执行以下检查：

1. **阅读清洗后的 20 份随机文档。** 它们是否残留 HTML、JavaScript、导航文本或样板文本（boilerplate）？如果是，说明你的 HTML 剥离步骤不完整。

2. **阅读通过质量过滤器的 20 份随机文档。** 其中是否包含垃圾信息、关键词列表或机器生成内容？如果是，请收紧过滤器阈值。

3. **阅读未通过质量过滤器的 20 份随机文档。** 其中是否包含真正的高质量内容？如果是，说明你的过滤器过于严格。请放宽阈值或为特定模式添加例外规则。

4. **阅读去重步骤中随机抽取的 20 对近似重复文档。** 它们实际上是否相似？如果不相似，请降低杰卡德（Jaccard）阈值或增加哈希函数数量。

## 数据混合比例

不存在通用的固定公式。请以以下基线为起点，并根据评估结果进行调整：

| 类别 | Llama 3 比例 | 起始参考点 |
|----------|--------------|----------------|
| 网页文本 | 50% | 50% |
| 代码 | 25% | 15-25% |
| 书籍/学术文献 | 13% | 10-15% |
| 数学 | 8% | 5-10% |
| 多语言网页 | 4% | 5-10% |

如果模型需要具备较强的编程能力，请提高代码比例。如果推理能力至关重要，请提高数学比例。如果需要降低数据噪声，请减少网页文本比例。每次调整比例后，务必在基准测试（benchmarks）上进行评估。

## 规模估算

针对给定的目标词元（token）数量：

- 1T 网页词元：预计需要约 3-5TB 原始文本，清洗和去重后约为 1.5-2TB
- 分词速度（Rust）：每核心约 1 亿词元/秒
- 分词速度（Python）：每核心约 100 万至 1000 万词元/秒
- 最小哈希（MinHash）去重（128 个哈希函数，16 个分带）：每核心约 1 万份文档/秒
- 序列打包：受 I/O 限制，对于 10GB 以上的语料库请使用内存映射文件（memory-mapped files）

对于 15T 词元（Llama 3 规模），请规划约 30-50TB 的原始输入数据、在 64 核机器上进行 1-2 周的预处理，以及 100TB 以上的磁盘空间用于存储中间文件。

## 训练前检查清单

1. 总词元（token）数量需与您的计算预算（compute budget）相匹配（可参考 Chinchilla 缩放定律（Chinchilla scaling）或 Llama 3 过训练比例（Llama 3 overtrain ratio）作为指导）
2. 去重（deduplication）已移除 30% 至 40% 的网络数据
3. 质量过滤（quality filter）已剔除剩余数据中的 10% 至 20%
4. 英文数据的压缩比（compression ratio）为 3 至 5 个字符/词元（chars/token）
5. 序列利用率（sequence utilization）需高于 95%
6. 随机抽查（spot-checks）表明，在数据处理流水线（pipeline）的每个阶段，文本均保持干净且连贯
7. 数据混合比例（data mix ratios）已在小规模训练运行（small-scale training run）中得到验证
8. 个人身份信息（PII）的移除已在样本中得到验证
9. 所有二进制格式（binary formats）（包括打包序列（packed sequences）和词元 ID 数组（token ID arrays））均通过了往返编解码测试（round-trip encoding/decoding tests）
10. 流水线具备可复现性（reproducibility）：在固定随机种子（fixed random seeds）的情况下，相同的输入将生成完全一致的输出
