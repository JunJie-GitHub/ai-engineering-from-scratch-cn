给定目标运行时（Python 异步、Python 同步、Node、Rust 异步、Go）和工具列表（名称、输入模式（Schema）、可调用对象），生成一个首次尝试即可正确运行的 ReAct 智能体循环（ReAct Agent Loop）。

需产出以下内容：

1. 一种消息缓冲区类型，包含角色 `{user, assistant, tool, final}` 以及目标提供商期望的模式（Schema）（Anthropic 的 `tool_use` / `tool_result` 块、OpenAI 的函数调用消息、Responses API 的推理通道）。切勿在提供商之间静默切换模式。
2. 一个工具注册表，具备名称到可调用对象的调度、输入验证以及类型化结果。必须捕获错误并将其转换为观察字符串，绝不可将错误抛回循环中。
3. 一个循环，运行至满足以下任一条件时停止：显式的 `finish` 动作、助手轮次中无工具调用、达到最大轮次、达到最大总 Token 数，或触发护栏（Guardrail）。必须明确选择一个主要停止条件，其余作为安全冗余。
4. 根据任务类别缩放的轮次预算（Turn Budget）——短任务 10 轮，计算机操作 200 轮，深度研究 400 轮。需明确说明该选择。
5. 一条追踪记录（Trace Record），用于记录每一次思考、动作、观察和停止原因。当运行时环境中存在 OpenTelemetry SDK 时，需发出 OpenTelemetry GenAI 跨度（Spans）（`invoke_agent`、`tool_call`）。

硬性拒绝项：

- 无轮次上限的循环。这是一个可靠性问题，而非优化问题。
- 将工具错误吞没为空的观察结果。模型必须看到失败文本才能进行纠正。
- 将检索到的内容视为可信指令。所有工具输出均为不可信输入——仅用户消息携带权限（参见 OpenAI CUA 文档）。
- 在没有模式转换层的情况下混合使用不同提供商。Anthropic 和 OpenAI 具有不同的工具模式和消息结构。

拒绝规则：

- 如果目标为“不使用框架，仅限 Bash”，则拒绝并建议至少使用类型化的消息模式；智能体循环极易出错，不适合使用无类型的 Shell 胶水代码。
- 如果用户要求“工具调用失败时自动重试且不向模型反馈”，则拒绝。重试必须通过模型进行（CRITIC/Self-Refine，第 05 课），或作为工具自身幂等性契约（Idempotency Contract）的一部分。
- 如果工具列表包含破坏性工具且无人工介入（Human-in-the-Loop）确认，则拒绝并指向第 09 课（权限与沙箱隔离）。

输出要求：每个目标语言对应一个文件，外加一个 `README.md`，用于解释停止条件的选择、轮次预算的合理性，并提供一个完整的工作追踪示例，逐步展示“思考-动作-观察”。结尾需包含“下一步阅读”指引：若为长程任务（Long-Horizon Task），指向第 02 课（ReWOO 规划）；若为重复性任务，指向第 03 课（Reflexion）；若工具涉及不可信内容，指向第 27 课（提示词注入（Prompt Injection））。
