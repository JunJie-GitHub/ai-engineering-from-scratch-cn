# AI Scientist v2 — 研讨会级别的自主研究

> Sakana 的 AI Scientist v2（Yamada 等人，arXiv:2504.08066）能够运行完整的研究循环（Research Loop）：提出假设、编写代码、执行实验、生成图表、撰写论文并提交。它是首个有生成论文通过 ICLR 2025 研讨会同行评审（Peer Review）的系统。独立评估（Beel 等人）发现，42% 的实验因编码错误而失败，且文献综述（Literature Review）经常将已有概念错误地标记为新颖成果。Sakana 官方文档警告称，该代码库会执行大语言模型（Large Language Model, LLM）编写的代码，并建议使用 Docker 隔离。这幅图景的两面正是其核心要义。

**类型：** 学习
**语言：** Python（标准库，研究循环状态机示例）
**前置要求：** 第 15 阶段 · 03 (AlphaEvolve)，第 15 阶段 · 04 (DGM)
**预计耗时：** 约 60 分钟

## 问题背景

研究是一项开放式任务。与 AlphaEvolve 的算法搜索或 DGM 受基准限制的自我修改不同，研究成果并没有机器可验证的正确性标准。论文是由评审专家而非单元测试（Unit Tests）来评判的。这使得该循环更难形成闭环——但一旦闭环，其价值也更大，因为研究正是实现累积式进步的源泉。

AI Scientist v1（Sakana, 2024）通过从人工编写的模板出发实现了闭环。大语言模型在固定框架内填充实验内容。AI Scientist v2（Yamada 等人, 2025）则通过结合智能体树搜索（Agentic Tree Search）与视觉语言模型（Vision-Language Model）批判循环（Critique Loop），移除了对模板的依赖。该系统能够生成想法、实现实验、产出图表、撰写论文，并根据评审反馈进行迭代。

同行评审结论：一篇由 v2 生成的论文被 ICLR 2025 研讨会接收（已作披露）。独立评估结论：该系统远未达到可靠水平。两者皆为事实。

## 核心概念

### 架构

1. **想法生成（Idea generation）。** 大语言模型（LLM）根据给定主题和已有文献提出研究想法。v1 版本使用模板；v2 版本则在假设空间中进行智能体搜索（agentic search）。
2. **新颖性检查（Novelty check）。** 文献检索步骤用于检查该想法是否已发表。Beel 等人的评估正是在此步骤发现了错误标注问题——已有方法经常被错误地归类为新颖。
3. **实验计划（Experiment plan）。** 智能体起草实验方案并编写代码。
4. **执行（Execution）。** 代码在沙盒（sandbox）中运行。失败结果会被反馈至重试循环（retry loop）。在 Beel 等人的测量中，此阶段有 42% 的实验因编码错误而失败。
5. **图表生成（Figure generation）。** 视觉语言模型（vision-language model）读取生成的图表并重写以提升清晰度。这是 v2 版本的关键技术新增项。
6. **论文撰写（Writeup）。** 大语言模型起草论文，并与内部审稿人进行迭代修改。
7. **可选：投稿（submission）。** 将论文提交至会议或期刊。

### 研讨会录用结果意味着什么

一篇由 v2 生成的论文通过了 ICLR 2025 研讨会的同行评审（peer review）。作者已向程序委员会披露了该论文的生成来源。此次录用仅是一个数据点，并不代表可以宣称该系统“能独立开展研究”。

重要背景：研讨会论文的录用门槛低于主会论文。同行评审本身存在噪声；在任何特定时期，都只有一小部分投稿会被录用。一次成功仅能证明概念可行（proof of concept），而非可靠性声明。2026 年发表在《Nature》上的论文记录了该端到端循环（end-to-end loop），且本身由人类研究人员共同撰写；它并非“系统独立撰写了一篇《Nature》论文”。

### 独立评估的发现

Beel 等人（arXiv:2502.14297）进行了一项外部评估。核心发现如下：

- **实验失败。** 42% 的实验因编码错误（如导入错误、张量形状不匹配、未定义变量）而失败。重试循环捕获了部分错误，但并非全部。
- **新颖性错误标注。** 文献检索步骤经常将已有概念标记为新颖。这在研究层面等同于幻觉（hallucination）。
- **呈现质量差距（Presentation-quality gap）。** 视觉语言模型的图表评估生成了出版级视觉效果，掩盖了底层实验的缺陷。

最后一点发现对当前阶段尤为重要。一个能产出令人信服的结果却未进行扎实研究的系统，比那些明显失败的系统更危险，而非更安全。评估必须深入到底层主张，而不能仅停留在图表层面。

### 沙盒逃逸（sandbox-escape）的担忧

Sakana 官方仓库的 README 警告称：

> 由于本软件会执行大语言模型生成的代码，我们无法保证其安全性。存在引入危险软件包、不受控的网络访问以及生成意外进程的风险。请自行承担使用风险，并建议采用 Docker 隔离。

这是在未经验证的领域中实现自主运行的典型操作形态。大语言模型编写代码；代码执行；代码可以执行该进程权限允许的任何操作。如果没有对文件系统、网络和进程操作进行硬性限制的沙盒，任何自主研究智能体都可能窃取数据、耗尽算力或重写自身。

AlphaEvolve 的沙盒管理相对简单，因为其评估器（evaluator）约束严格。而 AI Scientist v2 的循环运行的是目标开放的开放式代码。正因如此，它需要更强的隔离措施（最低要求为 Docker，推荐使用 seccomp / gVisor），并且在每次提交离开系统前必须进行人工审查。

### v2 在前沿技术栈中的位置

| 系统 | 目标 | 输出类型 | 评估器 | 已知缺陷 |
|---|---|---|---|---|
| AlphaEvolve | 算法 | 代码 | 单元测试 + 基准测试 | 受限于评估器的严格程度 |
| DGM | 智能体脚手架（agent scaffolding） | 代码 | SWE-bench | 奖励黑客攻击（reward hacking） |
| AI Scientist v2 | 研究论文 | 文本 + 代码 + 图表 | 同行评审（较弱） | 实验失败、错误标注、表面优化掩盖缺陷 |

在这三者中，v2 的自动评估器最弱，输出面最广，且距离公开成果的路径最短。目前的安全保障主要依赖于操作控制措施（沙盒、审查、披露）。

## 使用方法

`code/main.py` 将 v2 循环模拟为一个状态机（state machine）：构思 → 新颖性检查 → 实验 → 图表 → 撰写 → 评审 → 接受或迭代。每个状态都配置了失败概率，该概率源自 Beel 等人的研究发现。运行模拟器执行 N 次循环，并统计以下指标：

- 最终进入投稿阶段的构思数量。
- 包含被精修论文掩盖的关键实验缺陷的投稿数量。
- 重试预算（retry budget）在质量与产出率之间的权衡情况。

## 交付发布

`outputs/skill-ai-scientist-sandbox-review.md` 是一份双门控评审清单（two-gate review checklist），用于在研究循环智能体（research-loop agent）生成的任何内容离开沙盒（sandbox）前进行审查。

## 练习

1. 使用默认参数运行 `code/main.py`。有多少比例的循环运行能生成“干净”的论文？又有多少比例生成的论文存在实验失败缺陷，但被图表评审（figure critique）精修掩盖了？

2. 默认参数已采用 Beel 等人研究中的 42% / 25%。请分别使用 `--experiment-failure 0.20 --novelty-mislabel 0.10` 和 `--experiment-failure 0.60 --novelty-mislabel 0.40` 重新运行。在这两次运行中，“精修但存在缺陷”的论文占比发生了怎样的变化？

3. 阅读 Sakana AI Scientist v2 仓库 README 中关于沙盒（sandbox）要求的说明。列出你会为多日自主运行添加的两项额外限制（Docker 之外）。

4. 阅读 Beel 等人论文的第 4 节，了解呈现质量差距（presentation-quality gap）。设计一个额外的评估器（evaluator），用于识别那些外观精修但实验存在缺陷的论文。

5. 提出一套针对研究智能体（research-agent）输出的人工评审协议，使其扩展性优于“由一名博士阅读每篇论文”的模式。找出瓶颈并围绕其进行设计。

## 关键术语

| 术语 | 通常说法 | 实际含义 |
|---|---|---|
| AI Scientist v1 | “Sakana 的模板化研究智能体” | 将实验填充到固定框架中 |
| AI Scientist v2 | “无模板研究智能体” | 结合视觉语言模型（VLM）图表评审的智能体树搜索 |
| 智能体树搜索（Agentic tree search） | “分支型研究智能体” | 并行扩展多个实验计划；通过内部评审器进行剪枝 |
| 视觉语言评审（Vision-language critique） | “VLM 对图表进行精修” | 多模态模型读取图表并重写以提升清晰度 |
| 文献检索（Literature retrieval） | “新颖性检查” | 检索已有文献以确认构思的新颖性——已有记录表明其存在误标现象 |
| 精修掩盖（Polish masking） | “论文精美，研究存在缺陷” | 呈现质量高于实验质量；掩盖了研究弱点 |
| 沙盒逃逸（Sandbox escape） | “LLM 代码突破限制” | 智能体执行的代码产生了循环设计者未预期的行为 |
