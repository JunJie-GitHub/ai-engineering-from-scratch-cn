针对由 AI-Scientist-v2 风格循环（loop）生成的自主研究产出（假设、代码、实验、图表、论文草稿），需生成一份双门审查报告：沙盒审计（sandbox audit，检查是否有内容违规流出）与研究审计（research audit，评估研究工作是否严谨可靠）。

这两道门直接对应下方的审计项：**沙盒门（Sandbox gate）= 第 1 项**；**研究门（Research gate）= 第 2 项（实验审计）+ 第 3 项（润色审计）**。第 4–5 项规定了两道门均通过后后续的处理流程。

生成内容如下：

1. **沙盒门（Sandbox gate）。** 在任何产出物（artifact）离开沙盒前：
   - 列出循环发起的每一次网络调用及其目标地址。标记任何未经预先批准的调用。
   - 盘点循环在工作目录之外写入的所有文件。
   - 确认在整个运行期间，Docker / seccomp / gVisor 的容器隔离（containment）始终有效。
   - 确认没有子进程（subprocesses）脱离沙盒的监管。
   若任何检查项未通过，则阻止导出，并上报至人工处理。
2. **实验审计（Experiment audit）。** 审查实验代码，而非论文正文：
   - 验证所有声明的实验均已实际运行，且报告的数据具备可复现性。
   - 检查失败的实验是否如实报告为失败，而非事后重新包装为“阴性结果”。
   - 检查该想法的“新颖性（novelty）”标签是否经得起人类领域专家文献检索的验证。
3. **润色审计（Polish audit）。** 审查图表：
   - 确保每张图表的数据均来源于已记录的实验运行，而非润色阶段的重写或篡改。
   - 确认坐标轴、比例尺和注释与底层数据一致。
   - 标记任何图注（caption）声称超出数据支持范围的图表。
4. **披露计划（Disclosure plan）。** 若该产出物计划对外分发：
   - 披露该产出物由智能体（agent）生成。
   - 披露所使用的工具（模型家族、循环版本）。
   - 披露负责审查的人工审核员姓名及其审查的具体内容。
5. **否决发布决策（Negative-release decision）。** 若产出物未通过任何审计步骤，默认决策为不予发布。若要推翻此默认决策，必须由指定的人工负责人签字确认。

硬性拒绝条件（Hard rejects）：
- 任何跳过任一审查门的提交。
- 任何缺失或循环执行日志（execution logs）不完整的产出物。
- 任何无法追溯至特定实验运行的图表。
- 任何未经领域专家验证的新颖性声明。

拒绝规则（Refusal rules）：
- 若运行过程缺乏 Docker 或同等隔离机制，则予以拒绝，并要求在隔离沙盒中重新运行。
- 若用户无法提供实验阶段的执行日志，则予以拒绝——该论文无法进行审查。
- 若拟定的分发渠道为同行评审（peer-reviewed）场所，且用户提议不披露智能体作者身份，则予以拒绝，并要求强制披露。

输出格式：

返回一份双门审查报告：
- **沙盒门裁决（Sandbox gate verdict）**（通过 / 拦截，附理由）
- **研究门裁决（Research gate verdict）**（涵盖实验审计 (2) 与润色审计 (3)）（通过 / 拦截 / 需专家介入，附逐项检查备注）
- **披露计划（Disclosure plan）**（发布渠道、披露文本、人工审核员姓名）
- **发布决策（Release decision）**（发布 / 暂缓 / 拒绝）
- **下一步行动（Next action）**（明确责任人、具体任务及截止时间）
