# 特征选择 (Feature Selection) 策略

一份用于挑选并应用合适特征选择方法的快速参考指南。

## 步骤 1：从数据清理开始

在应用任何方法之前，先剔除明显无用的特征：

- **常量特征 (Constant Features)**：方差 = 0。直接移除。
- **近似常量特征 (Near-Constant Features)**：方差 < 0.01（或自定义阈值）。直接移除。
- **重复特征 (Duplicate Features)**：完全相同的列。保留其一，删除其余。
- **ID 列 (ID Columns)**：每行唯一，不包含可泛化的信息。直接移除。

此过程仅需数秒，在杂乱的真实世界数据集中通常可剔除 10% 到 30% 的特征。

## 步骤 2：根据实际情况选择方法

### 快速决策树

1. **特征数 < 50？** 从互信息 (Mutual Information) 排序开始。保留前 K 个。
2. **特征数 50 - 500？** 先使用方差阈值 (Variance Threshold) 过滤；若使用线性模型，接着采用 L1 正则化 (L1 Regularization) / Lasso；若使用树模型，则采用树重要性 (Tree Importance)。
3. **特征数 > 500？** 串联使用多种方法：方差阈值过滤 -> 互信息筛选（保留前 50%）-> 对剩余特征应用递归特征消除 (Recursive Feature Elimination, RFE)。
4. **需要可解释性？** L1 正则化能给出精确的零/非零系数。树重要性则提供排序得分。
5. **需要捕捉非线性关系？** 使用互信息或基于树的重要性。避免使用 L1（仅限线性）。
6. **需要考虑特征交互作用？** 使用 RFE 或基于树的重要性。过滤法 (Filter Methods) 会忽略交互作用。

### 方法参考

| 方法 | 适用场景 | 避免场景 |
|--------|------------|---------------|
| 方差阈值 | 始终作为第一步使用 | 切勿跳过 |
| 互信息 | 快速排序、非线性关系 | 需要检测特征交互作用时 |
| 递归特征消除 (RFE) | 精细筛选、特征数量适中 | 模型计算成本极高、特征数 > 1000 |
| L1 / Lasso | 线性模型、快速的嵌入式选择 (Embedded Selection) | 非线性问题、高度相关的特征 |
| 树重要性 | 非线性关系、特征交互作用 | 受高基数特征 (High-Cardinality Features) 偏差影响 |
| 置换重要性 (Permutation Importance) | 模型无关验证、最终检查 | 初始筛选时速度过慢 |

## 步骤 3：验证选择结果

- 对比使用所选特征与使用全部特征时的模型性能
- 使用交叉验证 (Cross-Validation)，而非单次训练集/测试集划分
- 若性能下降超过 1%~2%，说明可能误删了有用特征
- 若性能提升，说明成功剔除了噪声

## 步骤 4：处理常见陷阱

### 相关特征 (Correlated Features)
- L1 会从一组相关特征中随机挑选一个，并将其他特征的系数置零
- 先计算相关系数矩阵 (Correlation Matrix)，再决定保留哪些相关特征
- 树重要性会将重要性得分分散到相关特征上

### 数据泄露 (Data Leakage)
- 仅在训练数据上拟合特征选择器
- 将相同的选择逻辑应用于测试数据
- 在交叉验证中，特征选择必须在每个折 (Fold) 内部独立进行

### 对特征选择过程过拟合
- RFE 迭代次数过多可能导致对训练集过拟合
- 使用预留数据 (Held-out Data) 进行验证，而非用于选择的数据
- 使用稳定性选择 (Stability Selection，在子样本上重复执行) 以获得更稳健的结果

## 步骤 5：生产环境检查清单

- [ ] 已将方差阈值作为首要过滤器应用
- [ ] 特征选择仅在训练数据上进行拟合
- [ ] 已记录所选特征（名称、使用的方法、得分）
- [ ] 已对比性能：所选特征 vs 全部特征
- [ ] 已进行交叉验证，而非单次划分评估
- [ ] 特征选择已集成至训练流水线 (Training Pipeline) 中（非手动操作）
- [ ] 已部署特征漂移 (Feature Drift) 监控机制（所选特征可能随时间失效）
