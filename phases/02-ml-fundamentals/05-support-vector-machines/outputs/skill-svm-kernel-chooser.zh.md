# 支持向量机 (Support Vector Machine, SVM) 核函数选择指南

SVM 的性能主要由两个选择决定：核函数 (kernel)（决定决策边界的形状）和正则化参数 (regularization parameters)（控制间隔宽度与分类错误之间的权衡）。正确设置这些参数是区分无用模型与强大模型的关键。

## 决策检查清单

1. 数据是否线性可分（或接近线性可分）？
   - 是：使用线性核函数 (linear kernel)。它速度更快且更具可解释性。
   - 否：进入第 2 步。

2. 特征数量与样本数量的比例如何？
   - 特征远大于样本（例如使用 TF-IDF 的文本数据）：使用线性核函数。高维数据通常线性可分。使用径向基函数核 (Radial Basis Function, RBF) 只会增加复杂度而无实际收益。
   - 样本远大于特征（例如包含 10-50 个特征的表格数据）：RBF 核函数是默认选择。

3. 预期决策边界是否平滑？
   - 平滑、连续的边界：RBF 核函数
   - 多项式形状的边界：多项式核函数 (polynomial kernel)（建议从 2 次或 3 次开始）
   - 领域知识表明存在特定的交互项：使用匹配次数的多项式核函数

4. 数据集规模有多大？
   - 少于 10,000 个样本：任何核函数均可，RBF 是稳妥的默认选择
   - 10,000 到 100,000 个样本：线性核函数或 `LinearSVC`（原始形式，每轮时间复杂度为 O(n)）
   - 超过 100,000 个样本：不要使用基于核函数的 SVM。请改用线性 SVM、梯度提升 (gradient boosting) 或神经网络。

5. 是否对特征进行了缩放？
   - SVM 需要进行特征缩放。在拟合模型前务必进行标准化处理（零均值，单位方差）。未缩放的特征会扭曲间隔的几何结构。

如果 RBF 效果不佳，可尝试 2-3 次的多项式核函数。若仍然无效，则该问题可能不适合使用 SVM。

## 调优 C（正则化参数）

`C` 控制对误分类的惩罚力度。它与正则化强度成反比。

| C 值 | 效果 | 适用场景 |
|---------|--------|-------------|
| 0.001 - 0.01 | 间隔较宽，允许较多违反间隔的样本 | 数据噪声较大，追求泛化能力 |
| 0.1 - 1.0 | 均衡 | 良好的初始搜索范围 |
| 10 - 1000 | 间隔较窄，极少允许违反间隔 | 数据干净，需要高准确率 |

调优策略：
- 从 C=1.0 开始
- 在对数尺度上搜索：[0.001, 0.01, 0.1, 1, 10, 100, 1000]
- 使用交叉验证 (cross-validation) 选择最优值
- 如果最优 C 值位于搜索范围的边界，请沿该方向扩大搜索范围

## 调优 gamma（RBF 核函数）

`gamma` 控制单个训练样本的影响范围。它定义了高斯函数的宽度。

| gamma 值 | 效果 | 适用场景 |
|-------------|--------|-------------|
| 较小 (0.001) | 每个样本的影响范围较大。边界平滑、简单 | 欠拟合或特征较少时 |
| 中等 (auto: 1/n_features) | `sklearn` 默认值。合理的起点 | 通用场景 |
| 较大 (10+) | 每个样本仅影响邻近点。边界复杂、波动剧烈 | 存在过拟合风险时 |

调优策略：
- 从 gamma="scale"（即 1 / (n_features * X.var())，`sklearn` 默认值）开始
- 在对数尺度上搜索：[0.001, 0.01, 0.1, 1, 10]
- 低 gamma + 高 C 容易导致过拟合 (overfitting)
- 高 gamma + 低 C 容易导致欠拟合 (underfitting)

## 联合调优 C 和 gamma

`C` 和 `gamma` 存在交互作用。务必联合调优，切勿独立调整。

推荐方法：
1. 粗粒度网格搜索 (coarse grid search)：C 取值 [0.01, 0.1, 1, 10, 100]，gamma 取值 [0.001, 0.01, 0.1, 1, 10]（共 25 种组合）
2. 确定最优参数区域
3. 在最优区域周围进行细粒度网格搜索（例如 C 取值 [5, 10, 20, 50]，gamma 取值 [0.05, 0.1, 0.2]）
4. 全程使用 5 折交叉验证 (5-fold cross-validation)

## 常见错误

- 在高维稀疏数据上使用 RBF 核函数 (RBF kernel)（线性核函数效果更好且速度快 100 倍）
- 忘记进行特征缩放 (feature scaling)（这是支持向量机 (SVM) 最常见的错误）
- 在含噪声数据上将参数 C 设置过高（模型会记住噪声而非学习决策边界 (decision boundary)）
- 在样本量超过 5 万的数据集上使用核支持向量机 (kernel SVM)（训练时间将难以承受）
- 未同时调节 C 和 gamma 参数（两者会相互补偿）
- 默认使用 5 阶及以上的多项式核 (polynomial kernel)（极易导致严重过拟合，建议先尝试 2 或 3 阶）

## 快速参考

| 核函数 (Kernel) | 适用场景 | 关键参数 | 训练复杂度 |
|--------|------------|----------------|-------------------|
| 线性核 (Linear) | 文本/TF-IDF、特征数量多、数据量大 | 仅 C | 每轮 O(n) |
| RBF 核 | 通用场景、样本量低于 1 万 | C, gamma | O(n^2) 至 O(n^3) |
| 多项式核 (Polynomial) | 已知存在多项式关系的数据 | C, degree, coef0 | O(n^2) 至 O(n^3) |
| Sigmoid 核 | 极少适用（等效于两层神经网络） | C, gamma, coef0 | O(n^2) 至 O(n^3) |
