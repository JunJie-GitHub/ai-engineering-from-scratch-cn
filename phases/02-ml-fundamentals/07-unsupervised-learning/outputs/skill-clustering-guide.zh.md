# 聚类（Clustering）算法选择指南

聚类没有绝对最优的算法。正确的选择取决于簇的形状、是否已知簇的数量、数据中的噪声程度以及数据集的规模。

## 决策清单

1. 是否已知簇的数量？
   - 是：K-Means（K均值）或 GMM（高斯混合模型）
   - 否：DBSCAN（自动发现簇）或层次聚类（Hierarchical Clustering）（在不同层级切割树状图）

2. 簇的形状是怎样的？
   - 近似球形（团状）：K-Means
   - 椭圆形且大小不一：GMM
   - 任意形状（月牙形、环形、链状）：DBSCAN
   - 嵌套或层次结构：层次聚类

3. 数据中是否包含噪声或异常值？
   - 是：DBSCAN（显式标记噪声点）或 GMM（低概率点视为异常值）
   - 否：K-Means 即可

4. 是否需要软分配（Soft Assignments）（概率）？
   - 是：GMM 会为每个簇提供 P(簇 | 数据点) 的概率
   - 否：K-Means 或 DBSCAN 提供硬分配（Hard Assignments）

5. 数据集规模有多大？
   - 10,000 以下：任何算法均可
   - 10,000 到 1,000,000：K-Means（较快）、Mini-Batch K-Means（更快）
   - 超过 1,000,000：Mini-Batch K-Means 或 BIRCH。层次聚类速度过慢。

## 各算法适用场景

**K-Means**：默认的起点。速度快（时间复杂度为 O(n * k * iterations)），实现简单，且足以应对许多问题。可使用肘部法则（Elbow Method）或轮廓系数（Silhouette Score）来选择 K 值。局限性：假设簇为球形，对初始化敏感（建议使用 K-Means++ 或多次运行），难以很好地处理大小差异较大的簇。

**DBSCAN**：最适合发现任意形状的簇并自动检测异常值。包含两个参数：`eps`（邻域半径）和 `min_samples`（最小密度）。无需预先指定 K 值。局限性：当簇的密度差异很大时效果不佳，且调整 `eps` 参数较为棘手。可使用 k-距离图（k-distance plot）来估算 `eps`：计算每个点到其第 k 个最近邻的距离，排序后寻找拐点（elbow）。

**层次聚类（凝聚型）**：构建一棵合并树。当你希望在多个粒度上探索簇结构时非常有用（在不同高度切割树状图）。Ward 链接法（Ward's Linkage）最适合紧凑的簇。单链接法（Single Linkage）能发现拉长的簇，但对噪声敏感。局限性：内存复杂度为 O(n^2)，时间复杂度为 O(n^3)，因此不适用于大规模数据集。

**GMM**：基于概率分配的软聚类（Soft Clustering）。将每个簇建模为具有各自均值和协方差的高斯分布。当簇呈椭圆形或相互重叠时，效果优于 K-Means。可使用 BIC（贝叶斯信息准则）来选择组件数量。局限性：假设数据服从高斯分布，在非凸形状上可能失效，且对初始化敏感。

## 聚类质量评估（无标签情况）

| 指标 | 衡量内容 | 取值范围 | 适用场景 |
|--------|-----------------|-------|----------|
| 轮廓系数（Silhouette Score） | 簇内凝聚度与簇间分离度 | -1 到 1（越高越好） | 比较不同的 K 值或算法 |
| 惯性/簇内平方和（Inertia / within-cluster SS） | 簇的紧密程度 | 0 到无穷大（越低越好） | K-Means 的肘部法则 |
| BIC / AIC | 带复杂度惩罚的模型拟合度 | 越低越好 | 选择 GMM 的组件数量 |
| Calinski-Harabasz 指数 | 簇间方差与簇内方差的比值 | 越高越好 | 快速对比 |
| Davies-Bouldin 指数 | 簇间的平均相似度 | 越低越好 | 对重叠簇进行惩罚 |

## 常见误区

- 在未进行特征缩放（Feature Scaling）的情况下运行 K均值聚类（K-Means）（量纲较大的特征会主导距离计算）
- 当实际数据为高维时，仅凭肉眼观察二维数据来选择 K 值（应使用轮廓系数（Silhouette Score））
- 在非球形簇上使用 K均值聚类（K-Means）（月牙形或环形数据应使用基于密度的空间聚类（DBSCAN））
- 将 DBSCAN 的 `eps` 参数设置得过大（所有数据被归为一个簇）或过小（所有数据都被视为噪声）
- 将聚类标签视为真实标签（Ground Truth）（聚类属于探索性分析；需结合领域知识进行验证）
- 在包含超过 20,000 个数据点的数据集上运行层次聚类（Hierarchical Clustering）（内存消耗和计算时间会急剧增加）
