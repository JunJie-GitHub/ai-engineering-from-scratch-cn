# 回归策略指南

回归（Regression）用于预测连续值。选择合适的方法取决于特征（Features）与目标变量（Target）之间的关系、特征数量以及过拟合（Overfitting）的风险。

## 决策清单

1. 特征与目标变量之间的关系是否近似线性？
   - 是：从普通线性回归（Ordinary Linear Regression）开始
   - 否：尝试多项式特征（Polynomial Features）或非线性模型

2. 相对于样本量，特征数量有多少？
   - 特征少、样本多：普通线性回归即可胜任
   - 特征多、样本少：使用正则化（Regularization）方法（如岭回归 Ridge 或 Lasso 回归）
   - 特征多于样本：使用 Lasso（L1 正则化）进行特征选择，或使用岭回归（Ridge，L2 正则化）收缩所有权重

3. 是否需要模型可解释性（Interpretability）？
   - 是：使用特征较少的线性回归，或使用 Lasso 进行自动特征选择
   - 否：使用多项式特征，或转向基于树的模型（Tree-based Models）或神经网络（Neural Networks）

4. 数据集是否较小（少于 10,000 行）？
   - 为提升速度，使用正规方程（Normal Equation，闭式解）
   - 交叉验证（Cross-validation）对于可靠评估至关重要

5. 数据集是否较大（数百万行）？
   - 使用随机梯度下降（Stochastic Gradient Descent, SGD）或小批量梯度下降（Mini-batch Gradient Descent）
   - 正规方程因涉及 O(n^3) 复杂度的矩阵求逆而过于缓慢

## 各方法适用场景

**普通线性回归（Ordinary Linear Regression）**：任何回归任务的基线（Baseline）。建议从此入手。如果决定系数（R-squared）可接受且模型足够简单，则无需继续尝试其他方法。

**多项式回归（Polynomial Regression）**：散点图呈现曲线而非直线时使用。建议从 2 阶开始。仅在验证集性能提升时才增加阶数。阶数大于 5 几乎必然导致过拟合。

**岭回归（Ridge Regression, L2）**：适用于存在大量相关特征的场景。所有权重会向零收缩，但不会完全变为零。当你认为所有特征都对预测有贡献时，此方法效果较好。

**Lasso 回归（L1）**：特征众多且你怀疑仅有少数特征有效时使用。Lasso 会将无关特征的权重压缩至精确的零，从而实现自动特征选择。

**弹性网络（Elastic Net）**：结合 L1 和 L2 惩罚项。当你拥有大量相关特征且希望进行一定程度的特征选择时使用。

## 常见误区

- 在梯度下降前跳过特征缩放（Feature Scaling）（会导致收敛速度极慢）
- 使用测试集性能来调整超参数（Hyperparameters）（应使用验证集或交叉验证）
- 未检查验证误差就直接拟合高阶多项式（训练集 R^2 会随阶数增加而单调上升）
- 忽略残差图（Residual Plots）（若残差呈现规律性模式，R^2 可能会产生误导）
- 将 R^2 视为唯一评估指标（应同时检查残差分布、平均绝对误差 MAE 以及业务特定阈值）

## 快速参考

| 方法 | 适用场景 | 正则化方式 | 特征选择 |
|--------|------------|---------------|-------------------|
| 普通最小二乘法（OLS） | 基线模型，特征较少 | 无 | 手动 |
| 岭回归（Ridge） | 特征众多，且均相关 | L2（收缩权重） | 否 |
| Lasso 回归 | 特征众多，仅少数有效 | L1（置零） | 自动 |
| 弹性网络（Elastic Net） | 存在大量相关特征 | L1 + L2 | 部分 |
| 多项式回归（Polynomial） | 非线性关系 | 可叠加 Ridge/Lasso | 手动选择阶数 |
