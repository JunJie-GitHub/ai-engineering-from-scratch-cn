# 模型评估策略 (Model Evaluation Strategy)

这是一份用于正确评估任何机器学习 (Machine Learning, ML) 模型的清单。遵循此流程可避免最常见的评估错误。

## 步骤 1：正确划分数据 (Data Splitting)

- 在任何预处理（如缩放 (Scaling)、插补 (Imputation)、编码 (Encoding)）之前进行划分
- 分类任务使用分层划分 (Stratified Split)
- 预留一个测试集 (Test Set)，仅在最后阶段接触一次
- 对于小型数据集，使用 5 折或 10 折交叉验证 (Cross-Validation) 代替单次划分
- 对于时间序列数据，使用基于时间的划分（切勿打乱顺序）

## 步骤 2：选择合适的评估指标 (Evaluation Metric)

### 分类任务 (Classification)

| 场景 | 使用此指标 | 原因 |
|-----------|----------------|-----|
| 类别均衡，简单对比 | 准确率 (Accuracy) | 易于解释，在类别数量相等时具有实际意义 |
| 假阳性代价高昂（如垃圾邮件过滤、欺诈预警） | 精确率 (Precision) | 衡量被标记为阳性的样本中实际为阳性的比例 |
| 假阴性代价高昂（如癌症筛查、安防） | 召回率 (Recall) | 衡量实际阳性样本中被正确检出的比例 |
| 需要平衡精确率与召回率 | F1 分数 (F1 Score) | 调和平均数，对极端不平衡情况惩罚较重 |
| 跨阈值对比模型 | AUC-ROC | 独立于阈值的排序质量评估 |
| 类别不平衡数据 | F1、AUC-ROC 或 PR-AUC | 在类别不平衡时，准确率会产生误导 |

### 回归任务 (Regression)

| 场景 | 使用此指标 | 原因 |
|-----------|----------------|-----|
| 标准回归，可接受异常值 | 均方根误差 (RMSE) | 与目标变量单位一致，对较大误差惩罚更重 |
| 对异常值鲁棒的评估 | 平均绝对误差 (MAE) | 平等对待所有误差，不受异常值主导 |
| 在不同量纲上对比模型 | 决定系数 (R-squared) | 归一化至 0-1 范围（表示解释的方差比例） |
| 业务需要具体金额误差 | MAE 或 RMSE | 可直接解释为误差的实际大小 |

## 步骤 3：建立基线 (Baseline)

在评估模型之前，先计算基线性能：
- 分类任务：多数类预测器（始终预测出现频率最高的类别）
- 回归任务：始终预测训练集目标变量的均值
- 任何无法超越这些基线的模型都未学到有效规律

## 步骤 4：交叉验证 (Cross-Validation)

- 使用 K 折交叉验证（K=5 或 K=10）以获得稳定的估计值
- 分类任务使用分层 K 折交叉验证 (Stratified K-Fold)
- 报告各折结果的均值与标准差
- 均值为 0.85 且标准差为 0.02 的模型，比均值为 0.87 但标准差为 0.10 的模型更可靠

## 步骤 5：进行统计学模型对比

- 未经显著性检验，不要仅凭最高平均分选择模型
- 在交叉验证的各折结果上使用配对 t 检验 (Paired t-test)
- 若 |t| < 2.78（对应 K=5，自由度 df=4，p<0.05），则差异可能源于偶然
- 当性能差异不显著时，优先选择更简单的模型

## 步骤 6：排查常见错误

- 数据泄露 (Data Leakage)：测试集信息是否流入训练过程？（如划分前进行缩放、使用目标变量衍生的特征）
- 类别不平衡：准确率是否掩盖了少数类表现不佳的问题？
- 过拟合 (Overfitting)：训练集与验证集的性能差距是否过大？
- 评估次数过多：是否多次查看过测试集结果？

## 步骤 7：报告最终性能

- 使用训练集与验证集合并后的数据进行最终训练
- 仅在预留的测试集上评估一次
- 尽可能报告所选指标及其置信区间 (Confidence Interval)
- 说明与基线的对比情况（比随机猜测或均值预测提升多少）
