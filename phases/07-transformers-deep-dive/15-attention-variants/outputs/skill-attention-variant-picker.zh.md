# 注意力变体选择器（Attention Variant Picker）

帮助开发者为新的 Transformer（变换器）模型选择并论证合适的注意力拓扑结构，或为正在扩展至更长上下文的现有模型提供选型依据。

## 需收集的输入信息

1. **目标上下文长度**：训练时与推理时（通常不同——许多模型在 16K 长度下训练，在推理时进行扩展）。
2. **检索需求**（1–5 分制）：1 = 纯对话，5 = 大海捞针（Needle-in-a-Haystack）/ RAG（检索增强生成，Retrieval-Augmented Generation）/ 带有长仓库上下文的代码任务。
3. **推理内存预算**：单次请求的 KV 缓存（Key-Value Cache）容忍度（正确的单位是每层每 token 的字节数）。
4. **训练成本容忍度**：从头训练 SWA（滑动窗口注意力，Sliding Window Attention）成本较低；将差分注意力（Differential Attention）改造到预训练模型中则成本高昂。
5. **目标硬件**：Hopper+ 架构完整支持 FlashAttention-3，Ada 架构支持 FA2，较旧的 GPU 则受限于掩码计算（Mask-Limited）。

## 决策规则

- **上下文 ≤ 16K 且检索需求 ≤ 3**：使用结合 FlashAttention 的全量注意力（Full Attention）。切勿过早优化。
- **上下文 16–128K 且检索需求 ≤ 3**：采用 SWA 与全局注意力 5:1 混合模式，窗口大小设为 1024（参考 Gemma 3 架构）。在压缩 KV 缓存的同时保持检索能力可用。
- **上下文 > 128K**：采用全量 SWA，每 4–6 层插入一个全局注意力层，并结合位置插值（Position Interpolation）/ YaRN 缩放技术（参见第 04 课）。
- **检索需求 = 5 且训练预算允许**：仅在最顶层的 4 层考虑使用差分注意力（可将 KV 缓存翻倍量减半，同时获得大部分“注意力汇聚点抵消”的收益）。
- **对外提供公共 API**：优先选择稳定模式（全量、SWA、Gemma-3 混合）。除非团队有底层内核工程师，否则跳过原生稀疏（Native-Sparse）/ 差分注意力（DIFF）。
- **无法更改基础模型**：SWA 可通过掩码在推理阶段进行改造适配；而差分和稀疏注意力则无法做到。

## 始终需注意的警告

- 参数量低于 7B 的纯 SWA 模型在推理基准测试中通常会出现可衡量的性能下降。不建议采用。
- 窗口大小 < 512 几乎总是不合适的。请增大窗口或改用其他拓扑结构。
- 论文中关于差分注意力的报告均基于小型模型（3–7B）。截至 2026 年初，其在大规模扩展方面的证据仍然有限。
- 每种变体都会与 RoPE（旋转位置编码，Rotary Position Embedding）/ YaRN 缩放技术产生交互（参见第 04 课）。请明确声明所采用的位置编码方案。

## 输出格式

返回以下内容：

1. **推荐方案**：单一命名的拓扑结构（例如：“Gemma-3 混合模式，W=1024，SWA:全局 = 5:1”）。
2. **论证依据**：将各项输入与上述决策规则逐一对应。
3. **KV 缓存预估**：在目标上下文长度下，以“每层每 token 字节数”及“Batch Size 为 1 时的 GB 数”表示。
4. **迁移路径**：若基础模型已训练完成，说明如何进行改造适配。
5. **已知风险**：哪些基准测试 / 工作负载可能会出现性能回退。
